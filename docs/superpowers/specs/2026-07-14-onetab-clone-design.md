# sontab — OneTab 클론 크롬 익스텐션 설계 (v1)

날짜: 2026-07-14
상태: 승인됨 (TypeScript 적용 반영)

## 목표

OneTab의 기본 기능을 동일하게 제공하는 가벼운 크롬 익스텐션을 만든다.
v1은 OneTab 기능 동등(parity)이 목표이며, 부가기능은 v1 완성 후 별도 기획한다.

## 기술 스택

- **Manifest V3** 크롬 익스텐션
- **TypeScript 전면 적용**: 서비스 워커, 공용 모듈, Svelte 컴포넌트
  (`<script lang="ts">`) 모두 TS. `@types/chrome`으로 chrome API 타입 지원.
- **Svelte 5 + Vite**: 탭 목록 페이지 UI. Svelte는 컴파일 타임에 순수 JS로
  변환되어 런타임 오버헤드가 거의 없다 (번들 수 KB 수준).
- **빌드**: Vite 멀티 엔트리(페이지 + 서비스 워커) + `manifest.json`/아이콘
  정적 복사. CRXJS 등 익스텐션 전용 플러그인은 쓰지 않는다.
- **원칙**: 개발 의존성(devDependencies)은 무거워도 무방하다. 최적화 대상은
  **빌드 산출물(dist/)의 크기와 런타임 성능**이다. 런타임 의존성은 0을 목표.
- 백엔드 없음. 모든 데이터는 로컬 저장.

## 권한 (manifest.json)

- `tabs` — 탭 제목/URL 수집 및 닫기
- `storage` + `unlimitedStorage` — 목록 저장 (기본 5MB 제한 해제)
- 호스트 권한 없음

## v1 기능 범위

### 탭 수집 (핵심 동작)
1. 툴바 아이콘 클릭 → 현재 창의 탭을 모두 수집
2. 수집 제외 대상: 고정(pinned) 탭, 익스텐션 자신의 목록 페이지
3. 수집된 탭들은 하나의 "그룹"으로 목록 맨 위에 저장 (생성 시각 기록)
4. 수집된 원본 탭은 닫는다
5. 목록 페이지를 연다 (이미 열려 있으면 해당 탭으로 포커스)
6. 수집할 탭이 0개면 아무것도 저장하지 않고 목록 페이지만 연다

### 목록 페이지
- 그룹 목록: 최신 그룹이 위. 각 그룹에 탭 수, 생성 시각, 이름(선택) 표시
- 그룹 이름 인라인 편집
- 그룹 잠금 토글: 잠긴 그룹은 복원해도 항목이 제거되지 않고, 삭제 버튼 비활성화
- 탭 항목: 파비콘 + 제목 표시, 클릭 시 복원
- **개별 복원**: 새 탭으로 열고 목록에서 제거 (잠긴 그룹은 유지)
- **그룹 전체 복원**: 그룹의 모든 탭을 열고 그룹 제거 (잠긴 그룹은 유지)
- **개별 삭제 / 그룹 삭제** (잠긴 그룹은 삭제 불가)
- 전체 탭 수 요약 표시 (예: "탭 12개, 그룹 3개")

### 가져오기 / 내보내기
- 내보내기: `URL | 제목` 형식의 텍스트 (OneTab 호환), 그룹 사이 빈 줄
- 가져오기: 같은 형식의 텍스트를 붙여넣으면 그룹으로 추가

## 데이터 모델

`chrome.storage.local` 키 `groups`:

```json
{
  "groups": [
    {
      "id": "uuid",
      "createdAt": 1760000000000,
      "name": "",
      "locked": false,
      "tabs": [
        { "id": "uuid", "url": "...", "title": "...", "favIconUrl": "..." }
      ]
    }
  ]
}
```

- 정렬은 배열 순서 그대로 (새 그룹은 앞에 unshift)
- 파비콘은 탭의 `favIconUrl`을 그대로 저장. 없으면 기본 아이콘 표시

## 컴포넌트 구조

```
src/
  background.ts        # 서비스 워커: 아이콘 클릭 → 수집/저장/닫기/페이지 열기
  types.ts             # TabGroup, SavedTab 등 공용 타입
  storage.ts           # groups CRUD (get/save/addGroup/removeTab/...) — 공용 모듈
  list/
    index.html         # 목록 페이지 엔트리
    main.ts            # Svelte 마운트
    App.svelte         # 전체 레이아웃 + 상태
    Group.svelte       # 그룹 헤더(이름/잠금/복원/삭제) + 탭 목록
    TabItem.svelte     # 탭 한 줄
    ImportExport.svelte# 텍스트 가져오기/내보내기 패널
public/
  manifest.json
  icons/
```

- 상태 동기화: 목록 페이지는 `chrome.storage.onChanged`를 구독해 서비스
  워커가 저장한 변경을 즉시 반영한다 (페이지가 열린 채 수집해도 갱신됨).

## 에러 처리 / 엣지 케이스

- `chrome://` 등 특수 URL: 수집은 하되, 복원 실패 시 해당 항목을 목록에 유지
- 저장 실패(quota 등): 탭을 닫기 **전에** 저장을 완료하고, 저장 실패 시 탭을
  닫지 않는다 (데이터 유실 방지 — 저장 성공 후에만 닫기)
- 가져오기 텍스트 파싱 실패 줄은 건너뛰고 나머지만 추가

## 테스트

- `storage.ts`와 가져오기/내보내기 파서는 Vitest 단위 테스트
- 타입 검사: `svelte-check` + `tsc --noEmit`
- 수집/복원 플로우는 크롬에 로드해 수동 검증 (chrome API 목킹은 v1에서 과함)

## v2 방향 (사용자 비전, 2026-07-14 — 이번 범위 아님)

OneTab의 "무한히 내려가는 일자 리스트"와 차별화하는 것이 핵심:

- **패턴화/자동 분류**: 저장된 탭을 도메인·주제 기준으로 묶어서 보여준다
  (규칙 기반 → AI 분류로 발전 가능)
- **즐겨찾기**: "다시 읽을 페이지"를 별도로 관리 (복원해도 유지되는 항목)
- **태그**: 탭/그룹에 태그 부여 및 태그별 조회 (즐겨찾기 다음 단계)
- **AI 연동**: 저장된 탭 데이터(chrome.storage.local)는 익스텐션 코드가
  자유롭게 읽고 쓸 수 있으므로, 익스텐션 안에서 AI에게 제목/URL을 넘겨
  자동 분류·태깅·요약이 가능하다. 선택지: Chrome 내장 AI(Gemini Nano,
  온디바이스·무료), 외부 LLM API(Claude 등, 품질↑·API 키와 비용 필요),
  로컬 모델(Ollama + native messaging). 프라이버시(URL 외부 전송 여부)가
  선택 기준.

기타 후보: 드래그 앤 드롭 재정렬, 검색, 중복 탭 처리 옵션, 다크 모드,
단축키, 컨텍스트 메뉴("이 탭만 보내기" 등), 클라우드 백업/동기화, 공유.

**v1 데이터 모델과의 호환**: 위 기능들은 `SavedTab`/`TabGroup`에 선택적
필드(`starred?`, `tags?`, `category?`)를 추가하는 방식으로 확장한다.
기존 저장 데이터는 필드가 없으면 기본값으로 취급 — 마이그레이션 불필요.
